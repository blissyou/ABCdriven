### **펫픽 (PetPick) 앱 개발 요구사항 명세서**

**문서 버전: 1.0**
**작성일: 2026-01-20**

#### **1. 개요**
본 문서는 데이터 기반 맞춤형 펫케어 추천 커머스 플랫폼 '펫픽(PetPick)'의 성공적인 개발을 위해 필요한 기능 및 시스템 요구사항을 정의합니다. 기획서 내용을 바탕으로 각 요구사항을 식별하고, 중요도, 유형, 상세 설명 및 성공 기준을 명시합니다.

---

#### **2. 기능 요구사항 (Functional Requirements)**

| ID | 요구사항 명 | 상세 설명 | 중요도 | 유형 | 성공 기준 (Acceptance Criteria) |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **FR-01** | **사용자 회원가입 및 로그인** | 사용자가 이메일, 소셜 계정 등을 통해 서비스에 가입하고 로그인할 수 있는 기능을 제공합니다. | 필수 | 기능 | - 이메일/비밀번호 기반 회원가입이 가능하다.<br>- 주요 소셜 계정(카카오, 네이버 등)을 이용한 간편 가입/로그인이 가능하다.<br>- 비밀번호 찾기 및 재설정 기능이 작동한다. |
| **FR-02** | **반려동물 프로필 등록 및 관리** | 보호자가 자신의 반려동물 정보를 등록, 수정, 삭제할 수 있는 기능을 제공합니다. | 필수 | 기능 | - 프로필 생성 시 기본 정보(종, 품종, 나이, 체중, 성별, 중성화 여부) 입력이 가능하다.<br>- 알러지 유발 성분을 텍스트 또는 선택지로 입력할 수 있다.<br>- 현재 앓고 있는 질환이나 특이사항을 기재할 수 있다.<br>- 한 명의 사용자가 여러 반려동물 프로필을 생성하고 전환하며 관리할 수 있다. |
| **FR-03** | **건강 데이터 입력 및 연동** | 동물병원 검진 기록, 웨어러블 데이터 등 외부 건강 데이터를 프로필에 통합하는 기능을 제공합니다. | 필수 | 기능 | - 사용자가 수동으로 검진 기록(주요 수치, 진단명 등)을 입력할 수 있다.<br>- 검진 기록 파일을 (PDF, 이미지 형식) 업로드하고 시스템이 인식할 수 있다.<br>- (선택/확장) 펫 웨어러블 기기의 API를 연동하여 활동량, 수면 데이터를 자동으로 가져올 수 있다. |
| **FR-04** | **맞춤형 제품 추천** | 등록된 반려동물 프로필을 기반으로 최적의 펫케어 제품(사료, 간식 등)을 추천하는 핵심 로직입니다. | 필수 | 기능 | - 프로필의 나이, 체중, 건강 상태를 기반으로 제품을 필터링하여 추천한다.<br>- 프로필에 등록된 **알러지 유발 성분이 포함된 제품은 추천 목록에서 반드시 제외**된다.<br>- 각 추천 제품에 대해 반려동물과의 '적합도 점수'가 명확하게 표시된다.<br>- 사용자가 '적합도 점수'의 상세 산출 근거(어떤 정보가 반영되었는지)를 확인할 수 있다. |
| **FR-05** | **제품 성분 정보 제공** | 추천된 제품의 성분 정보를 표준화된 형식으로 제공하여 사용자가 쉽게 이해할 수 있도록 합니다. | 필수 | 기능 | - 모든 제품의 전체 성분 정보가 통일된 형식으로 표시된다.<br>- 주요 성분에 대해 반려동물에게 미치는 영향(예: 유익, 주의, 위험)에 대한 설명이 제공된다.<br>- 특정 성분이 현재 반려동물의 건강 상태(질환, 알러지)와 어떤 관련이 있는지 하이라이트된다. |
| **FR-06** | **제품 가격 비교 및 구매 연결** | 제휴된 여러 쇼핑몰의 제품 가격을 비교하고, 사용자가 최적의 조건으로 구매할 수 있도록 연결합니다. | 필수 | 기능 | - 동일 제품에 대해 여러 제휴 쇼핑몰의 판매 가격과 배송비가 리스트업된다.<br>- 최저가 순으로 자동 정렬되며, 다른 기준으로도 정렬이 가능하다.<br>- '구매하기' 버튼 클릭 시 해당 쇼핑몰의 상품 페이지로 정확히 이동(딥링크)된다. |
| **FR-07** | **추천 피드백 수집** | 추천된 제품을 구매하고 사용한 경험에 대한 사용자 피드백을 수집하여 추천 알고리즘을 고도화합니다. | 필수 | 기능 | - 구매한 제품에 대해 별점(만족도) 평가가 가능하다.<br>- 제품 사용 후 반려동물의 반응(기호성, 변 상태 변화 등)을 객관식 또는 단문으로 기록할 수 있다.<br>- 피드백 데이터가 해당 사용자의 다음 추천 시 가중치로 반영된다. |
| **FR-08** | **건강 트렌드 리포트** | 수집된 반려동물의 건강 데이터(체중, 활동량 등) 변화 추이를 시각화하여 리포트로 제공합니다. | 선택 | 기능 | - 체중, 활동량, 음수량 등의 변화를 주간/월간 단위 그래프로 확인할 수 있다.<br>- 정기적으로(예: 매월 초) 요약된 건강 리포트가 알림 형태로 제공된다. |

#### **3. 비기능적 요구사항 (Non-functional Requirements)**

| ID | 요구사항 명 | 상세 설명 | 중요도 | 유형 | 성공 기준 (Acceptance Criteria) |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **NFR-01** | **개인정보보호 및 보안** | 사용자 및 반려동물의 민감 정보를 관련 법규에 따라 안전하게 보호하고 처리해야 합니다. | 필수 | 보안 | - 모든 개인 식별 정보 및 건강 데이터는 전송/저장 시 암호화된다.<br>- 개인정보처리방침을 명확히 고지하고, 데이터 활용에 대한 사용자 동의를 받는다.<br>- 개인정보보호법 등 관련 법률 및 규제를 준수한다. |
| **NFR-02** | **성능 및 응답성** | 사용자가 서비스를 이용할 때 지연이나 끊김 없는 쾌적한 경험을 제공해야 합니다. | 필수 | 성능 | - 프로필 기반 추천 결과가 3초 이내에 로딩되어야 한다.<br>- 제휴 쇼핑몰의 가격 정보는 최소 1일 1회 이상 업데이트되어 최신성을 유지해야 한다.<br>- 앱 내 페이지 전환 및 스크롤이 부드럽게 이루어져야 한다. |
| **NFR-03** | **사용성 및 접근성** | IT 기기 사용에 익숙하지 않은 사용자도 쉽게 서비스를 이용할 수 있도록 직관적인 UI/UX를 제공합니다. | 필수 | 사용성 | - 최초 가입 시 최소 정보만으로 시작할 수 있는 단계적 온보딩을 제공한다.<br>- 모든 핵심 기능은 3뎁스(Depth) 이하의 탐색으로 접근 가능하다.<br>- 텍스트 크기, 버튼 크기 등은 W3C 웹 접근성 가이드라인을 준수한다. |
| **NFR-04** | **신뢰성 및 데이터 정확성** | 제공되는 모든 정보(제품 성분, 가격, 추천 근거)는 정확하고 신뢰할 수 있어야 합니다. | 필수 | 신뢰성 | - 추천의 근거가 되는 수의학적 정보나 영양학적 정보의 출처를 명시한다.<br>- "본 추천은 수의사의 진단을 대체할 수 없습니다"와 같은 **의료 면책 조항을 명확히 고지**한다.<br>- 제품 정보(성분, 단종 여부)에 대한 정기적인 검증 및 업데이트 프로세스를 갖춘다. |
| **NFR-05** | **확장성 및 유지보수성** | 향후 새로운 기능(신규 웨어러블 연동, 추천 카테고리 확장 등)을 쉽게 추가할 수 있는 구조로 설계합니다. | 권장 | 아키텍처 | - 추천 알고리즘, 제품 DB, 사용자 프로필 등 주요 모듈이 독립적으로 개발/배포될 수 있다.<br>- 제휴 쇼핑몰 추가 시 기존 코드에 미치는 영향을 최소화하는 구조를 가진다.<br>- 코드 내 주석과 문서를 통해 다른 개발자가 쉽게 이해하고 유지보수할 수 있다. |
